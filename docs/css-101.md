# Introduction to CSS

CSS (Cascading Style Sheets) controls the **visual presentation** of HTML. While HTML provides structure, CSS adds colour, layout, typography, and responsiveness.

## How CSS Works

You write CSS *rules* that select HTML elements and apply styles to them:

```css
selector {
  property: value;
}
```

For example:

```css
p {
  color: navy;
  font-size: 1rem;
}
```

This turns all `<p>` text navy and sets the font size to `1rem`.

## Adding CSS to a Page

There are three ways to include CSS:

**1. External stylesheet (recommended)**

```html
<link rel="stylesheet" href="styles.css">
```

Place this inside the `<head>` of your HTML file. Keeps style separate from structure.

**2. Internal `<style>` block**

```html
<head>
  <style>
    body {
      background-color: #f5f5f5;
    }
  </style>
</head>
```

**3. Inline styles**

```html
<p style="color: red;">Warning!</p>
```

Avoid inline styles for anything beyond quick experiments — they are hard to maintain.

## Selectors

Selectors determine which elements a rule applies to:

| Selector | Example | Matches |
|---|---|---|
| Type | `h1` | All `<h1>` elements |
| Class | `.card` | Elements with `class="card"` |
| ID | `#header` | Element with `id="header"` |
| Descendant | `nav a` | `<a>` elements inside a `<nav>` |
| Pseudo-class | `a:hover` | An `<a>` when hovered |

```css
/* Type selector */
h1 { font-size: 2rem; }

/* Class selector */
.card { border: 1px solid #ccc; }

/* ID selector */
#logo { width: 120px; }
```

Prefer classes over IDs for styling — classes are reusable.

## The Box Model

Every HTML element is a rectangular box. The box model describes the space it occupies:

```
┌─────────────────────────────┐
│           margin            │
│  ┌───────────────────────┐  │
│  │        border         │  │
│  │  ┌─────────────────┐  │  │
│  │  │     padding     │  │  │
│  │  │  ┌───────────┐  │  │  │
│  │  │  │  content  │  │  │  │
│  │  │  └───────────┘  │  │  │
│  │  └─────────────────┘  │  │
│  └───────────────────────┘  │
└─────────────────────────────┘
```

```css
.box {
  width: 300px;
  padding: 16px;
  border: 2px solid #333;
  margin: 24px;
}
```

Use `box-sizing: border-box` to include padding and border in the stated width:

```css
*, *::before, *::after {
  box-sizing: border-box;
}
```

## Colours and Typography

```css
body {
  color: #333;                  /* text colour */
  background-color: #fff;       /* background */
  font-family: sans-serif;      /* font stack */
  font-size: 16px;              /* base size */
  line-height: 1.5;             /* line spacing */
}

a {
  color: #0066cc;
}

a:hover {
  color: #004499;
  text-decoration: underline;
}
```

### CSS Units

| Unit | Description | Example |
|---|---|---|
| `px` | Absolute pixels | `border: 1px solid` |
| `%` | Relative to parent | `width: 50%` |
| `rem` | Relative to root font size | `font-size: 1.25rem` |
| `em` | Relative to current font size | `margin: 0.5em` |
| `vw` / `vh` | Viewport width / height | `height: 100vh` |

## Layout with Flexbox

Flexbox makes it easy to arrange items in a row or column:

```css
.container {
  display: flex;
  gap: 16px;
  justify-content: space-between; /* horizontal alignment */
  align-items: center;            /* vertical alignment */
}
```

Common `justify-content` values: `flex-start`, `flex-end`, `center`, `space-between`, `space-around`.

## Layout with Grid

CSS Grid is ideal for two-dimensional layouts:

```css
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 24px;
}
```

This creates a three-column layout where each column takes an equal share of available space.

## Responsive Design with Media Queries

Media queries apply styles only when certain conditions (such as screen width) are met:

```css
/* Default: single column */
.grid {
  display: grid;
  grid-template-columns: 1fr;
}

/* Wider screens: two columns */
@media (min-width: 600px) {
  .grid {
    grid-template-columns: 1fr 1fr;
  }
}
```

Design *mobile-first* — write base styles for small screens and use `min-width` media queries to progressively enhance for larger screens.

## Specificity

When multiple rules target the same element, CSS uses *specificity* to decide which one wins:

1. Inline styles (highest)
2. ID selectors
3. Class / attribute / pseudo-class selectors
4. Type selectors (lowest)

If two rules have equal specificity, the one that appears *later* in the stylesheet wins.
